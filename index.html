01 / framing

Mulai dari masalah, bukan dari bab.

Pembaca umum tidak perlu masuk lewat Bab I. Mereka perlu tahu mengapa topik skripsi ini layak diperhatikan, siapa yang terdampak, dan pertanyaan apa yang ingin dijawab penelitian.

Slot isi nanti: judul skripsi, latar belakang singkat, objek penelitian, dan satu kalimat "mengapa ini penting" dalam bahasa non-akademik.

02 / method

Metode dibuat seperti peta kerja.

Bagian metodologi sebaiknya tidak tampil sebagai paragraf panjang. Untuk project interaktif, metode lebih kuat jika dipresentasikan sebagai urutan keputusan: sumber data, cara memilih sampel, teknik analisis, lalu batasan riset.

Input [Dataset / responden / dokumen]
Filter [Kriteria pilihan data]
Analyze [Metode analisis]
Limit [Batasan penelitian]

Interaksi yang disarankan: setiap kartu metode bisa dibuka untuk menampilkan definisi, rumus, atau contoh kecil. Default-nya tetap ringkas agar tidak terasa seperti PDF dipindahkan mentah.

03 / evidence

Data diberi ruang, bukan dekorasi.

Karena isi final belum dimasukkan, area visual di bawah dibuat sebagai placeholder jujur. Nanti bisa diganti dengan chart, tabel ringkas, peta konsep, atau screenshot instrumen penelitian.

Prinsipnya: satu layar hanya membawa satu bukti utama. Jika ada banyak tabel, pecah menjadi beberapa momen baca agar pembaca tidak merasa sedang membuka lampiran.

05 / model storyboard

Storyboard menjembatani skripsi dan demo.

Sebelum pembaca melihat sinyal, mereka perlu tahu asal keputusan model. Storyboard ini membuat pipeline riset terlihat sebagai rangkaian keputusan yang bisa diaudit.

1

Ambil data harga emiten energi.

Data historis dipakai sebagai bahan mentah. UI cukup menampilkan cakupan waktu, sumber data, dan daftar emiten yang ikut diuji.

OHLCV IDX energy 2015-2025
2

Ubah harga menjadi indikator teknikal.

Bollinger Bands, Stochastic, OBV, ADX, dan ATR menjadi bahasa antara data pasar dan model. Ini lebih mudah dipahami jika divisualkan sebagai lapisan fitur.

ADX OBV ATR
3

Bentuk label BUY, HOLD, SELL.

Label sebaiknya dijelaskan sebagai aturan penelitian, bukan janji profit. Di sini pembaca melihat kapan data disebut naik, netral, atau turun.

3-class ATR rule horizon
4

Latih SVM dengan validasi waktu.

Untuk demo publik, bagian ini harus menampilkan bahwa data masa depan tidak ikut mengajari model masa lalu. Split waktu adalah bagian dari desain kepercayaan.

SVM time split gap check
5

Bandingkan sinyal dengan backtest.

Output akhir bukan hanya sinyal hari ini. Pembaca juga perlu melihat performa historis, confusion matrix, dan kapan model gagal.

backtest confusion risk note
